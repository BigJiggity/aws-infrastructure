# Plan 01-01: Pre-flight Script + State Bootstrap

**Phase:** 01-Foundation
**Requirements:** FOUND-01, FOUND-04, STATE-01, STATE-02
**Goal:** Operator can validate the existing org for CT blockers and bootstrap the S3 + DynamoDB state backend, with Terragrunt root config wired to OpenTofu.

## Pre-conditions

- [ ] AWS CLI is installed and configured with management account credentials (`aws sts get-caller-identity` returns the management account ID)
- [ ] The caller identity has OrganizationsFullAccess, IAMReadOnlyAccess, ConfigReadOnly, and CloudTrailReadOnly to complete pre-flight checks
- [ ] The caller identity has S3FullAccess and DynamoDBFullAccess in the management account to bootstrap state
- [ ] `tofu` 1.12.2 is installed (`tofu --version`)
- [ ] `terragrunt` 1.0.8 is installed (`terragrunt --version`)

## Tasks

### Task 1: Write and validate `scripts/preflight.sh`

**Files:**
- `scripts/preflight.sh` — pre-flight validation script; hard-blocks on any CT enrollment blocker

**Steps:**

1. Create directory `scripts/` at repo root.

2. Create `scripts/preflight.sh` with the following structure and behavior:

   **Header / preamble:**
   ```
   #!/usr/bin/env bash
   set -euo pipefail
   ```
   Define ANSI color codes: `RED='\033[0;31m'`, `GREEN='\033[0;32m'`, `YELLOW='\033[1;33m'`, `NC='\033[0m'`.
   Define helper functions `pass()`, `fail()`, `info()` that print colored `[PASS]`, `[FAIL]`, `[INFO]` prefixes.
   Maintain a `BLOCKERS=0` counter; increment on every `fail()` call.
   At script end, if `BLOCKERS -gt 0`, print a summary count and `exit 1`.

   **Check 1 — AWS Config recorders (per D-08):**
   Run: `aws configservice describe-configuration-recorders --query 'ConfigurationRecorders[*].name' --output text`
   If any recorders are returned, call `fail()` and print remediation:
   ```
   Remediation:
     aws configservice stop-configuration-recorder --configuration-recorder-name <name>
     aws configservice delete-delivery-channel --delivery-channel-name <name>
     aws configservice delete-configuration-recorder --configuration-recorder-name <name>
   ```
   If none, call `pass()`.

   **Check 2 — Conflicting SCPs (per D-08):**
   Run: `aws organizations list-policies --filter SERVICE_CONTROL_POLICY --query 'Policies[*].{Id:Id,Name:Name}' --output json`
   For each SCP, check if it is attached to the root:
   `aws organizations list-targets-for-policy --policy-id <id>`
   If any SCP attached to the root explicitly Denies any of `controltower:*`, `cloudformation:*`, `config:*`, or `cloudtrail:*`, call `fail()` with remediation note: "Detach or update the SCP before CT setup. CT requires FullAWSAccess at root." Otherwise call `pass()`.
   Note: This check inspects SCP JSON via `aws organizations describe-policy --policy-id <id> --query 'Policy.Content'`. Parse with `jq`. If `jq` is not installed, warn but do not block.

   **Check 3 — IAM role conflicts (per D-08):**
   Run: `aws iam get-role --role-name AWSControlTowerExecution 2>&1`
   If the role exists, verify it has a trust to the management account and `AdministratorAccess` policy. If it exists with wrong trust/policy, call `fail()` with remediation: "Delete and let CT recreate: `aws iam delete-role --role-name AWSControlTowerExecution`"
   If role does not exist, call `pass()` — CT will create it.
   Also check for `AWSControlTowerAdmin`, `AWSControlTowerCloudTrailRole`, `AWSControlTowerConfigRecorderRole`. If any exist, call `fail()` with remediation: "These are CT-managed roles. Delete each: `aws iam delete-role --role-name <name>`"

   **Check 4 — Organizations trusted access (per D-08):**
   Run: `aws organizations list-aws-service-access-for-organization --query 'EnabledServicePrincipals[*].ServicePrincipal' --output json`
   Check that `controltower.amazonaws.com`, `config.amazonaws.com`, `cloudtrail.amazonaws.com`, and `sso.amazonaws.com` are all present.
   For each missing principal, call `fail()` with remediation: `aws organizations enable-aws-service-access --service-principal <principal>`
   All present → `pass()`.

   **Check 5 — CloudTrail conflicts (per D-08):**
   Run: `aws cloudtrail describe-trails --include-shadow-trails false --query 'trailList[*].{Name:Name,IsMultiRegionTrail:IsMultiRegionTrail,HomeRegion:HomeRegion}' --output json`
   If any multi-region trail exists in us-east-1, call `fail()` with remediation: "CT creates its own mandatory multi-region trail. Disable or delete the existing trail to avoid conflicts: `aws cloudtrail delete-trail --name <name>`"
   No multi-region trail in us-east-1 → `pass()`.

3. Make the script executable: `chmod +x scripts/preflight.sh`

4. The script must be idempotent — safe to run multiple times; it only reads, never mutates.

**Verification:**
- [ ] `bash --norc -n scripts/preflight.sh` exits 0 (syntax check passes)
- [ ] Running `scripts/preflight.sh` with management account credentials completes all 5 checks and prints colored output
- [ ] On a clean org, exits 0; if a mock failure is injected (e.g., by temporarily adding a fake Config recorder name to the describe output), the script exits 1 with the correct remediation message
- [ ] Script does not modify any AWS resources — only reads

### Task 2: Write `scripts/bootstrap-state.sh` and `terragrunt.hcl` root config

**Files:**
- `scripts/bootstrap-state.sh` — creates S3 state bucket and DynamoDB lock table via `aws` CLI (per D-05, D-06)
- `terragrunt.hcl` — repo-root Terragrunt config wiring `terraform_binary = "tofu"` and shared `remote_state` (per D-01, D-02, STATE-01, STATE-02)
- `management/state-bootstrap/` — placeholder directory confirming D-01 flat layout; no HCL needed here (state is bootstrapped via CLI script)

**Steps:**

1. Create `scripts/bootstrap-state.sh`:

   **Header / preamble:**
   ```
   #!/usr/bin/env bash
   set -euo pipefail
   ```
   Resolve management account ID at runtime:
   ```
   ACCOUNT_ID=$(aws sts get-caller-identity --query Account --output text)
   BUCKET_NAME="chain-vote-tofu-state-${ACCOUNT_ID}"
   TABLE_NAME="chain-vote-tofu-locks"
   REGION="us-east-1"
   ```

   **Create S3 bucket (idempotent):**
   Check if bucket exists: `aws s3api head-bucket --bucket "${BUCKET_NAME}" 2>&1`
   If exit code 0, print "[INFO] Bucket ${BUCKET_NAME} already exists — skipping creation."
   If exit code non-zero (bucket does not exist or access denied), create it:
   ```
   aws s3api create-bucket \
     --bucket "${BUCKET_NAME}" \
     --region "${REGION}"
   ```
   Note: us-east-1 does NOT use `--create-bucket-configuration` (that flag is only for other regions).

   Enable versioning:
   ```
   aws s3api put-bucket-versioning \
     --bucket "${BUCKET_NAME}" \
     --versioning-configuration Status=Enabled
   ```

   Enable SSE-S3 encryption:
   ```
   aws s3api put-bucket-encryption \
     --bucket "${BUCKET_NAME}" \
     --server-side-encryption-configuration \
       '{"Rules":[{"ApplyServerSideEncryptionByDefault":{"SSEAlgorithm":"AES256"}}]}'
   ```

   Block all public access:
   ```
   aws s3api put-public-access-block \
     --bucket "${BUCKET_NAME}" \
     --public-access-block-configuration \
       "BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true"
   ```

   **Create DynamoDB table (idempotent):**
   Check if table exists: `aws dynamodb describe-table --table-name "${TABLE_NAME}" --region "${REGION}" 2>&1`
   If exit code 0, print "[INFO] DynamoDB table ${TABLE_NAME} already exists — skipping creation."
   If exit code non-zero, create it:
   ```
   aws dynamodb create-table \
     --table-name "${TABLE_NAME}" \
     --attribute-definitions AttributeName=LockID,AttributeType=S \
     --key-schema AttributeName=LockID,KeyType=HASH \
     --billing-mode PAY_PER_REQUEST \
     --region "${REGION}"
   ```
   Wait for table to become ACTIVE:
   ```
   aws dynamodb wait table-exists --table-name "${TABLE_NAME}" --region "${REGION}"
   ```

   Print a final summary block:
   ```
   echo ""
   echo "=== State Backend Bootstrap Complete ==="
   echo "S3 Bucket : ${BUCKET_NAME}"
   echo "DynamoDB  : ${TABLE_NAME}"
   echo "Region    : ${REGION}"
   echo ""
   echo "Add to your environment or save these values:"
   echo "  export TF_STATE_BUCKET=${BUCKET_NAME}"
   echo "  export TF_STATE_LOCK_TABLE=${TABLE_NAME}"
   ```

2. Make the script executable: `chmod +x scripts/bootstrap-state.sh`

3. Create `management/state-bootstrap/` directory with a `.gitkeep` file. This directory is the placeholder for D-01's flat layout; state resources are created by the script above, not by `tofu apply` (chicken-and-egg per D-05).

4. Create `terragrunt.hcl` at repo root per D-02:

   ```hcl
   # Root Terragrunt configuration
   # All child units inherit this via include {} block.

   # Explicitly pin OpenTofu binary (Pitfall 9: non-deterministic binary detection)
   # Terragrunt v1.x defaults to "tofu" but explicit pin makes intent unambiguous (STATE-01).
   terraform_binary = "tofu"

   # Shared remote state configuration (STATE-02).
   # State key is derived from the calling unit's path relative to this file,
   # so no manual key management is required.
   remote_state {
     backend = "s3"
     generate = {
       path      = "backend.tf"
       if_exists = "overwrite_terragrunt"
     }
     config = {
       bucket         = "chain-vote-tofu-state-${get_aws_account_id()}"
       key            = "${path_relative_to_include()}/terraform.tfstate"
       region         = "us-east-1"
       encrypt        = true
       dynamodb_table = "chain-vote-tofu-locks"
     }
   }

   # Generate provider version constraints inherited by all child units.
   generate "versions" {
     path      = "versions.tf"
     if_exists = "overwrite_terragrunt"
     contents  = <<EOF
   terraform {
     required_version = ">= 1.8, < 2.0"
     required_providers {
       aws = {
         source  = "hashicorp/aws"
         version = ">= 6.0.0, < 7.0.0"
       }
     }
   }
   EOF
   }

   # Generate provider configuration inherited by all child units.
   # Units that need additional providers (e.g., multi-account aliases) override locally.
   generate "provider" {
     path      = "provider.tf"
     if_exists = "skip"
     contents  = <<EOF
   provider "aws" {
     region = "us-east-1"
   }
   EOF
   }
   ```

   Key decisions encoded here:
   - `terraform_binary = "tofu"` satisfies STATE-01 and guards against Pitfall 9.
   - `path_relative_to_include()` satisfies STATE-02 — no manual key strings.
   - `get_aws_account_id()` satisfies D-06 — bucket name embeds management account ID at runtime.
   - `if_exists = "skip"` on the provider generate block lets child units override it without conflict.

**Verification:**
- [ ] `bash --norc -n scripts/bootstrap-state.sh` exits 0
- [ ] Running `scripts/bootstrap-state.sh` creates the S3 bucket `chain-vote-tofu-state-<management_account_id>` and DynamoDB table `chain-vote-tofu-locks` in us-east-1
- [ ] Running `scripts/bootstrap-state.sh` a second time prints "[INFO] ... already exists" for both resources and exits 0
- [ ] S3 bucket has versioning enabled, SSE-S3 encryption enabled, public access blocked: verify with `aws s3api get-bucket-versioning`, `get-bucket-encryption`, `get-public-access-block`

## Threat Model

| Risk | Mitigation |
|------|-----------|
| S3 bucket name collision (`chain-vote-tofu-state-<account_id>` already taken by another account in same org) | Account ID suffix makes collisions effectively impossible; bootstrap script checks for existing bucket before creating |
| State bucket accidentally public | `put-public-access-block` with all four flags set to true; verified in post-conditions |
| Concurrent bootstrap runs creating duplicate DynamoDB table | Script checks for table existence before creating; `aws dynamodb wait table-exists` ensures readiness before printing success |
| Pre-flight script with elevated read permissions used to exfiltrate org structure | Script is read-only; does not write credentials or policy content to disk; review before running in sensitive envs |
| SCP check uses `jq` which may not be installed | Script emits a warning and skips the SCP content parse (check 2) without failing; operator must manually inspect SCPs if `jq` absent |
| Terragrunt auto-creates state bucket if bootstrap script was not run first | `get_aws_account_id()` in root `terragrunt.hcl` uses the correct naming convention, so if auto-create fires it will produce the correct bucket name; versioning/encryption will NOT be applied — documented as a MUST-NOT-SKIP warning in the runbook |
